Hailo calibration data

Hailo Calibration Data Generator

This module creates calibration datasets for Hailo quantization processes. It processes images from a source directory, validates them, and prepares them in the format required for Hailo’s neural network quantization workflow.

The script performs the following operations:

  1. Searches for image files in the specified directory

  2. Validates image integrity

  3. Resizes and crops images to target dimensions

  4. Saves processed images in the calibration directory

Example

Basic usage from command line:

$ python hailo_calibration_data.py --data_dir /path/to/images --target_dir calib_data

With custom parameters:

$ python hailo_calibration_data.py --data_dir /path/to/images \
    --target_dir my_calib --image_size 416 416 --num_images 512

Note

The script maintains aspect ratio during resizing and uses center cropping to achieve the target dimensions. Images that are too small or corrupted are automatically skipped.

Requirements:

  • PIL (Pillow)

  • tqdm

  • Standard library modules (os, argparse, random)

Author: PFG Electronics Training Team Date: June 2025

hailo_calibration_data.resize_and_crop(img, target_w, target_h)[source]

Resize and crop image to target size while maintaining aspect ratio.

This function performs intelligent resizing by first scaling the image to ensure at least one dimension matches the target, then center-cropping to achieve the exact target dimensions. The aspect ratio is preserved during resizing.

Parameters:

  • img (PIL.Image.Image) – Input PIL Image object to be processed

  • target_w (int) – Target width in pixels

  • target_h (int) – Target height in pixels

Returns:

Processed image with exact target dimensions

Return type:

PIL.Image.Image

Raises:

  • ValueError – If image has invalid dimensions (0 width or height)

  • ValueError – If image is too small to resize properly

  • ValueError – If calculated crop bounds are invalid

Example

>>> from PIL import Image
>>> img = Image.open('input.jpg')
>>> resized_img = resize_and_crop(img, 640, 640)
>>> print(resized_img.size)
(640, 640)

Note

The function uses LANCZOS resampling for high-quality resizing and performs center cropping to maintain the most important image content.

hailo_calibration_data.find_image_files(directory, extensions)[source]

Find all image files with specified extensions in directory and its subdirectories.

This function recursively traverses the given directory and all its subdirectories to locate image files that match the specified file extensions. The search is case-insensitive for file extensions.

Parameters:

  • directory (str) – Root directory path to search for images

  • extensions (list) – List of file extensions to search for (e.g., [‘.jpg’, ‘.png’])

Returns:

List of absolute file paths to all found image files

Return type:

list

Example

>>> extensions = ['.jpg', '.jpeg', '.png']
>>> image_files = find_image_files('/dataset/images', extensions)
>>> print(f"Found {len(image_files)} images")
Found 1500 images

Note

The function performs case-insensitive extension matching, so it will find files with extensions like ‘.JPG’, ‘.PNG’, ‘.jpeg’, etc.

hailo_calibration_data.validate_images(image_paths)[source]

Return only valid images (non-corrupt) from the provided list of image paths.

This function validates each image by attempting to open and verify it using PIL. Corrupted or unreadable images are automatically filtered out with appropriate error messages displayed to the user.

Parameters:

image_paths (list) – List of file paths to image files to validate

Returns:

List of file paths to valid, non-corrupted images

Return type:

list

Example

>>> all_images = ['/path/img1.jpg', '/path/corrupted.jpg', '/path/img3.png']
>>> valid_images = validate_images(all_images)
Validating images...
Skipping invalid image /path/corrupted.jpg: cannot identify image file
>>> print(f"Valid images: {len(valid_images)}")
Valid images: 2

Note

The validation process uses PIL’s verify() method which checks image integrity without fully loading the image into memory, making it efficient for large datasets.

hailo_calibration_data.main()[source]

Main execution function for the Hailo calibration data generator.

This function orchestrates the entire calibration data generation process: 1. Parses command line arguments 2. Creates output directory 3. Searches for and validates images 4. Processes images to target specifications 5. Saves processed images for Hailo quantization

The function handles various error conditions gracefully and provides informative feedback to the user throughout the process.

Raises:

SystemExit – If no images are found or no valid images remain after validation

Example

The main function is called when the script is executed directly:

$ python hailo_calibration_data.py --data_dir /dataset --num_images 512
